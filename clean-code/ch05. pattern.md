
# 1.  **“상수는 하위 유틸 함수 내부에 하드코딩하지 말고, 호출하는 상위 함수에서 넘겨주는 것이 더 명확하다.”** 

## **❌ 나쁜 예 (Temporal Coupling + 하드코딩된 상수)**
```csharp
private string GetPageNameOrDefault(HttpRequest request)
{
    var pageName = request.Query["page"];
    if (string.IsNullOrWhiteSpace(pageName))
        pageName = "HomePage"; // ← 하드코딩된 상수!
    return pageName;
}
```
### **문제점:**

- "HomePage"라는 **기본값이 왜 들어갔는지 문맥이 없음**
- GetPageNameOrDefault()는 재사용 불가능 (무조건 “HomePage”)
- 호출자가 제어할 수 없음 → **책임이 뒤바뀜**
## **✅ 좋은 예 (의도를 드러내는 상위 전달 방식)**
```csharp
public IActionResult HandleRequest(HttpRequest request)
{
    string pageName = GetPageNameOrDefault(request, "HomePage");
    var html = RenderPage(pageName);
    return Content(html, "text/html");
}

private string GetPageNameOrDefault(HttpRequest request, string defaultPageName)
{
    var pageName = request.Query["page"];
    return string.IsNullOrWhiteSpace(pageName) ? defaultPageName : pageName;
}
```
### **장점:**

| **항목**      | **설명**                                     |
| ----------- | ------------------------------------------ |
| **문맥 노출**   | “HomePage”가 기본값이라는 의도가 ==상위 함수에서 명확하게 보임== |
| **함수 재사용성** | ==default 값을 다르게 해서 다른 곳에서도 재사용 가능==       |
| **책임 분리**   | ==기본값 결정은 상위 계층의 책임==, 로직은 하위 계층의 책임       |
