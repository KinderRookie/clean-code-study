# 1. 감싸기
### Bad: 각 예외에 대해 반복되고 같은 대응 구조
```csharp
var port = new ACMEPort(12);

try
{
    port.Open();
}
catch (DeviceResponseException e)
{
    ReportPortError(e);
    _logger.Log("Device response exception", e);
}
catch (ATM1212UnlockedException e)
{
    ReportPortError(e);
    _logger.Log("Unlock exception", e);
}
catch (GMXError e)
{
    ReportPortError(e);
    _logger.Log("Device response exception", e);
}
finally
{
    // cleanup logic
}
```

### Good: 호출하는 라이브러리 API 감싸기
```csharp
var port = new LocalPort(12);

try
{
    port.Open();
}
catch (PortDeviceFailureException e)
{
    ReportError(e);
    _logger.Log(e.Message, e);
}
finally
{
    // cleanup logic
}
```

```csharp
public class LocalPort
{
    private readonly ACMEPort _innerPort;

    public LocalPort(int portNumber)
    {
        _innerPort = new ACMEPort(portNumber);
    }

    public void Open()
    {
        try
        {
            _innerPort.Open();
        }
        catch (DeviceResponseException e)
        {
            throw new PortDeviceFailureException(e);
        }
        catch (ATM1212UnlockedException e)
        {
            throw new PortDeviceFailureException(e);
        }
        catch (GMXError e)
        {
            throw new PortDeviceFailureException(e);
        }
    }
}
```
-> 외부 API를 사용하는 경우 유용함, 의존성 줄어듦

# 2.  정상적인 상황을 정의하라 - 기본값을 처리하기

- 일반적으로는 위에서 봤던 방식들이 유용하지만, catch문에서 예외적인 상황(special case)을 처리해야 하는 경우 코드가 더러워지는 일이 발생할 수 있다.
- 이런 경우, Martin Fowler의 Special Case Pattern을 사용하자.
- 예외 발생 시 catch문에서 처리하는 대신 ==**“정상 객체처럼 보이는 대체 객체”**== 를 리턴하여 ==**상위 호출자가 예외를 신경 쓰지 않아도 되도록 리팩터링**== 하는 방법입니다.

```csharp
try
{
    var expenses = expenseReportDAO.GetMeals(employee.Id);
    total += expenses.GetTotal();
}
catch (MealExpensesNotFoundException)
{
    total += GetMealPerDiem();
}
```
### **문제점:**
- GetMeals()가 실패할 가능성을 항상 고려해야 함
- try-catch 반복 발생
- 호출자는 예외 상황을 **비즈니스 로직과 섞어서 처리**해야 함 → 가독성 저하

### 해결
`expenseReportDAO.GetMeals()` 을 수정하여 언제나 `IMealExpenses` 를 반환하도록 수정
대체 객체(Special Case Object) `PerDiemMealExpenses` 를 반환.
```csharp
public class PerDiemMealExpenses : IMealExpenses
{
    public int GetTotal()
    {
        return GetMealPerDiem();
    }

    private int GetMealPerDiem()
    {
        return 25; // 디폴트 식비
    }
}
```
#### 호출부
```csharp
var expenses = expenseReportDAO.GetMeals(employee.Id);
total += expenses.GetTotal(); // 예외 신경 안 써도 됨!
```
